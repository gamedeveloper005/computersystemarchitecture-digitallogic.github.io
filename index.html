DL · 01 / 16Digital Logic

Number Systems & Representation

Frequency

How signed integers, decimals and fractions are stored in fixed-width registers — the basis for arithmetic, overflow, and floating-point questions.

Conversions between binary, octal, decimal and hexadecimal are assumed knowledge — NET rarely tests these directly. What it tests is how negative numbers and fractions are represented, because that's where the subtle, examiner-favourite traps live.

Signed Representation Schemes

For an n-bit word, three classical schemes exist. The differences between their ranges and zero-handling are asked constantly.

SchemeNegative formRange (n bits)Zeros
Sign-magnitudeMSB = sign flag, rest = magnitude−(2ⁿ⁻¹−1) to +(2ⁿ⁻¹−1)2 (+0, −0)
1's complementFlip every bit−(2ⁿ⁻¹−1) to +(2ⁿ⁻¹−1)2 (+0, −0)
2's complementFlip every bit, then add 1−2ⁿ⁻¹ to +(2ⁿ⁻¹−1)1

Worked example — represent −15 in 8-bit 2's complement

15 = 00001111  → invert every bit (1's complement) → 11110000  → add 1 → 11110001

So −15 is stored as 11110001. Verify by decoding it the same way: invert → 00001110, add 1 → 00001111 = 15, with the original sign bit confirming it's negative.

Common Trap — the asymmetric range

For 8-bit 2's complement the range is −128 to +127 — not symmetric. The pattern 10000000 represents −128, which has no positive counterpart. Negating −128 (invert + 1) returns 10000000 again — this is the textbook 2's-complement overflow case examiners love to ask about.

BCD & Related Codes

BCD (8421 code): each decimal digit is stored in its own 4-bit binary field, 0000–1001 for 0–9. The patterns 1010–1111 are invalid in pure BCD.

Excess-3: add 3 to the decimal digit before converting to binary. This makes the code self-complementing — the 9's complement of a digit is obtained by simply inverting every bit, which simplifies BCD subtraction circuits. This "why is Excess-3 self-complementing" angle is a recurring question.

Gray code: successive values differ in exactly one bit. Used to avoid glitches in counters/ADCs and forms the cell-ordering rule for Karnaugh maps (see DL·03).

IEEE-754 Floating Point

FormatTotal bitsSignExponentMantissaBias
Single precision321823127
Double precision64111521023
Key Formula
bias = 2^(k − 1) − 1,   where k = number of exponent bits

This is how 127 (k=8) and 1023 (k=11) are derived — if a question changes the exponent width, re-derive the bias from this formula rather than recalling a memorised number.

Worked example — encode 10.5 (single precision)

10.5 in binary = 1010.1 → normalise → 1.0101 × 2³

Sign = 0 (positive)  |  Exponent = 3 + 127 = 130 = 10000010  |  Mantissa = 01010000000000000000000 (the digits after the leading 1, padded to 23 bits)

Full 32-bit pattern: 0 10000010 01010000000000000000000

Exam Focus

Know the special-case exponent patterns: all-zero exponent → the number is zero or denormalised (mantissa gives the value); all-one exponent → infinity (mantissa = 0) or NaN (mantissa ≠ 0). "What does this bit pattern represent?" questions hinge entirely on these two cases.

Boolean Algebra & Logic Gates

Frequency

The algebra and gate behaviours that every later circuit-design question is built from.

Laws & Theorems

Exam Focus

De Morgan's theorem is the single most-used simplification tool in NET — it's how "implement using only NOR gates" or "convert this AND-OR expression to NAND-only" questions are solved. A useful extra identity worth remembering: A + A'B = A + B.

Logic Gate Truth Table

One table, every 2-input gate. Memorise this as a single block — most truth-table and "identify the gate from its output" questions reduce to reading this table.

ABANDORNANDNORXORXNOR

NOT: single input, output is the inverse (0→1, 1→0). Quick recall: XOR = output 1 when inputs differ; XNOR = output 1 when inputs are the same.

Universal Gates

NAND and NOR are each universal — every other gate can be built from either one alone. For NAND:

  • NOT — tie both inputs together: A NAND A = A'. (1 gate)
  • AND — a NAND followed by a NAND-as-NOT: (A NAND B)' = AB. (2 gates)
  • OR — invert A and B first (2 gates), then NAND the results: by De Morgan's, (A'·B')' = A+B. (3 gates total)

NOR works symmetrically (NOR-as-NOT, then NOR-NOR for OR, then invert for AND). Expect "minimum gates required" questions built directly on these constructions — we'll draw the actual circuits on DL·04.

Canonical Forms

A minterm is a product term containing every variable (true or complemented) that equals 1 for exactly one input row; a maxterm is its dual for sums, equalling 0 for exactly one row.

  • SOP (Sum of Products) — OR of minterms where the function output is 1, written as Σ(minterms).
  • POS (Product of Sums) — AND of maxterms where the function output is 0, written as Π(maxterms).

K-maps (DL·03) exist purely to turn these long canonical forms into a minimal SOP or POS expression.

Key Formula
Number of distinct Boolean functions of n variables = 2^(2ⁿ)

For n=2 → 16 functions; for n=3 → 256 functions. This exact phrasing — "how many Boolean functions of n variables exist" — appears almost every cycle.

DL · 03 / 16Digital Logic

Karnaugh Maps & Minimization

Frequency

A grid-based shortcut for turning a long canonical SOP/POS expression into its minimal form.

Cell Adjacency & Layout

A K-map places one cell per minterm so that physically adjacent cells differ in exactly one variable — which is why row/column headers run in Gray code order (00, 01, 11, 10), not binary order (00, 01, 10, 11). The map also wraps top-to-bottom and left-to-right: the first and last row/column — and all four corners — are mutually adjacent.

Worked Example — 4-variable map with don't-cares

F(A,B,C,D) = Σm(1,3,5,7,9) + d(11,13,15) — required minterms are plotted as 1; don't-cares (X) may be read as 0 or 1, whichever minimizes better.

AB \ CD00011110
000110
010110
110XX0
1001X0

The two middle columns (CD=01 and CD=11) are entirely 1s and X's — an octet (group of 8) where the only variable that stays constant across all eight cells is D=1. Reading the don't-cares as 1 lets this octet form, collapsing the function to F = D. Without the don't-cares you'd be left with smaller groups and a longer expression — exactly why don't-cares matter.

Grouping Rules

  • Groups must contain exactly 2ᵏ cells (1, 2, 4, 8, 16…) — never 3, 5, 6, or 7.
  • A group of 2ᵏ cells eliminates k variables from the term — bigger groups give shorter terms.
  • Groups can wrap around any edge of the map.
  • Every required 1 must be covered by at least one group; overlapping groups are allowed and often necessary.
  • Aim for the minimum number of largest possible groups that cover every 1.
Exam Focus

A prime implicant is a group that cannot be enlarged further. An essential prime implicant covers at least one minterm that no other prime implicant covers — it must appear in the final expression. "How many essential prime implicants does this function have?" is a recurring question type.

SOP vs POS Minimization

SOP (group the 1s): each group becomes a product (AND) term containing only the variables that stay constant across the group — uncomplemented if the variable is 1 throughout the group, complemented if it's 0. The minimized expression is the OR of these terms.

POS (group the 0s): the dual process — group the 0-cells the same way, but each group becomes a sum (OR) term where a variable that is 1 throughout the group appears complemented, and one that is 0 appears uncomplemented. The minimized expression is the AND of these sum terms.

Common Trap

Corner cells of a K-map are adjacent to each other (the map wraps like a torus). Forgetting wrap-around groupings is the single most common reason a "minimized" SOP/POS still matches a non-minimal distractor option.

DL · 04 / 16Digital Logic

Combinational Circuits

Frequency

Circuits whose output depends only on present inputs — adders, multiplexers, decoders, comparators.

Adders & Subtractors

CircuitInputsOutput equations
Half AdderA, BSum = A⊕B  |  Carry = AB
Full AdderA, B, CinSum = A⊕B⊕Cin  |  Cout = AB + B·Cin + A·Cin
Half SubtractorA, BDiff = A⊕B  |  Borrow = A'B
Full SubtractorA, B, BinDiff = A⊕B⊕Bin  |  Bout = A'B + A'·Bin + B·Bin

A full adder is two half adders plus an OR gate. Chaining n full adders (ripple-carry) builds an n-bit adder — the cumulative carry-propagation delay (= n × per-stage delay) is itself a common numerical question.

Multiplexers & Demultiplexers

Key Formula
A MUX with n select lines has 2ⁿ data inputs and 1 output

Demultiplexers mirror this: 1 input, n select lines, 2ⁿ outputs. So a 4:1 MUX needs 2 select lines; an 8:1 MUX needs 3.

Exam Focus

A 2ⁿ:1 MUX can implement any n-variable Boolean function: tie the n variables to the select lines, and set each data input D₀…D₂ⁿ₋₁ to the function's truth-table output for that combination. "Implement F(A,B,C) using a MUX" questions are solved by reading the truth table straight into the data inputs.

Decoders & Encoders

A decoder takes n inputs and activates exactly one of 2ⁿ outputs (one-hot) — e.g. a 3-to-8 decoder. An encoder is the reverse: 2ⁿ inputs compress to n outputs. When multiple encoder inputs can be active simultaneously, a priority encoder outputs the code for the highest-priority active input and ignores the rest.

Comparators & Code Converters

A magnitude comparator takes two n-bit numbers and produces three outputs: A>B, A=B, A<B. Cascading comparators (feeding the equality output into the next stage) extends the comparison to larger numbers.

Binary → Gray: the MSB is copied unchanged; each subsequent Gray bit = XOR of the current and previous binary bit (Gᵢ = Bᵢ ⊕ Bᵢ₊₁). Gray → Binary reverses this: the MSB is copied, then each binary bit = XOR of the current Gray bit and the previously computed binary bit.

DL · 05 / 16Digital Logic

Sequential Circuits

Frequency

Circuits with memory — output depends on present inputs and current state. The building block for every register and counter.

Latch vs Flip-Flop

A latch is level-triggered — it changes output as long as its enable/control input is active. A flip-flop is edge-triggered — it changes output only on a clock edge, giving synchronous, predictable timing. Every register and counter in a CPU is built from flip-flops, not raw latches.

Flip-Flop Types

TypeBehaviourCharacteristic equation
SRS=1,R=0 → Set  | S=0,R=1 → Reset  | S=R=0 → hold  | S=R=1 → invalidQ(next) = S + R'Q  (SR=0)
DOutput simply follows D on the clock edgeQ(next) = D
JKLike SR, but J=K=1 → toggle (fixes SR's invalid state)Q(next) = JQ' + K'Q
TT=0 → hold  | T=1 → toggleQ(next) = T⊕Q
Common Trap

The SR=1,1 invalid state and the JK race-around condition get confused. SR's S=R=1 is undefined. JK's J=K=1 is well-defined (toggle) — but if the clock pulse stays high longer than the propagation delay, the output can toggle repeatedly within one pulse. This race-around is solved by master-slave construction, not by avoiding J=K=1.

Master-Slave Configuration

Two flip-flops in series: the master is enabled while the clock is high (capturing the input), and the slave is enabled while the clock is low (transferring the master's output to the final output). Because master and slave are never enabled simultaneously, the output changes only once per clock cycle — eliminating race-around in JK flip-flops.

Excitation Tables

Given the present state Q and desired next state Q(next), the excitation table gives the inputs needed — the core tool for "design a sequential circuit using ___ flip-flops" questions.

QQ(next)SRDJKT
000X00X0
011011X1
10010X11
11X01X00

The D-column is just Q(next) itself — D flip-flops need no "design" step, which is why they're the default choice for registers.

DL · 06 / 16Digital Logic

Registers & Counters

Frequency

Groups of flip-flops that store/shift data, or step through a repeating sequence of states.

Shift Registers

TypeMeaning
SISOSerial In, Serial Out — data shifts in and out one bit at a time
SIPOSerial In, Parallel Out — serial-to-parallel conversion
PISOParallel In, Serial Out — parallel-to-serial conversion
PIPOParallel In, Parallel Out — acts as a simple buffer register

Counter Types

Asynchronous (ripple) counter: each flip-flop's clock input is driven by the previous flip-flop's output, so state changes ripple through the chain. Simple, but total propagation delay is the sum of every flip-flop's delay.

Synchronous counter: every flip-flop shares the same clock; extra combinational logic decides which flip-flops toggle. Faster (delay = one flip-flop's delay regardless of size) but needs more logic.

Key Formula
n flip-flops → MOD-2ⁿ counter (counts 0 to 2ⁿ−1)

For a MOD-N counter where N is not a power of 2 (e.g. a decade/MOD-10 counter), you need ⌈log₂N⌉ flip-flops plus reset/feedback logic that forces the counter back to 0 after N−1.

CounterConstructionStates (n FFs)
Ring counterShift register, output fed back to input — a single 1 circulatesn
Johnson (twisted-ring) counterLike a ring counter, but the complemented output is fed back2n
Exam Focus

"How many flip-flops for a ring counter with N states" and "how many states does an n-flip-flop Johnson counter have" are asked as a pair almost every cycle — keep the n vs 2n distinction crisp.

DL · 07 / 16Digital Logic

Computer Arithmetic

Frequency

How the ALU actually performs subtraction, multiplication and division on signed binary numbers.

Subtraction via 2's Complement

A − B is computed as A + (2's complement of B), so the ALU only needs an adder — no separate subtractor circuit. Any carry out of the MSB is simply discarded.

Key Formula — Overflow Detection
Overflow = Carry into MSB ⊕ Carry out of MSB

Equivalently: overflow happens only when adding two numbers of the same sign produces a result with the opposite sign. Adding numbers of different signs can never overflow.

Booth's Algorithm

Booth's algorithm multiplies two signed 2's-complement numbers directly, with no separate sign handling. It scans the multiplier two bits at a time — the current bit Qₙ and the previous bit Qₙ₋₁ (initialised to 0) — and decides the operation on the accumulator (AC) before an arithmetic shift right of AC:Q:

Qₙ Qₙ₋₁Operation on AC (before shift)
0 0No operation — shift only
0 1AC = AC + Multiplicand
1 0AC = AC − Multiplicand
1 1No operation — shift only

After n cycles (n = bits in the multiplier), the register pair AC:Q holds the product. The "01 → add, 10 → subtract" mapping is the single fact most Booth's-algorithm questions hinge on.

Division — Restoring vs Non-Restoring

Restoring division: subtract the divisor from the partial remainder. If negative, set the quotient bit to 0 and add the divisor back ("restore") before shifting; if non-negative, set the quotient bit to 1 and shift directly.

Non-restoring division: skips the restore step — if the previous remainder was negative, it adds the shifted divisor on the next step instead of subtracting, and vice versa. Faster on average since no extra add-back cycle is needed.

Control Unit Design

Frequency

How the CPU generates the timing and control signals that drive every other unit.

Hardwired vs Microprogrammed

AspectHardwiredMicroprogrammed
ImplementationFixed combinational logic + sequencerControl signals stored as microinstructions in a control store
SpeedFasterSlower (extra control-store access)
FlexibilityHard to modify — needs redesigning logicEasy to modify — rewrite microcode
Best suited toSmall, simple instruction setsLarge, complex instruction sets (CISC)

Horizontal vs Vertical Microprogramming

Horizontal: every bit in the control word maps directly to one control signal — wide words, no decoding needed, faster but uses more control-store space.

Vertical: control signals are encoded compactly and require a decoder to expand them into actual signals — narrower words, more compact, but decoding adds delay.

Exam Focus

Know the two key registers of a microprogrammed control unit: the Control Address Register (CAR), holding the address of the next microinstruction, and the Control Data/Buffer Register, holding the microinstruction currently being executed.

CA · 11 / 16Computer Architecture

Memory Hierarchy & Cache

Frequency

The richest source of numerical questions in this entire unit — mapping techniques, hit ratios, average access time.

The Hierarchy & Why It Works

Registers → Cache (L1/L2/L3) → Main Memory → Secondary Storage: speed and cost-per-bit fall as capacity rises. The whole hierarchy is justified by locality of reference — temporal locality (recently accessed data is likely accessed again soon) and spatial locality (data near a recently accessed address is likely accessed soon too).

Cache Mapping Techniques

TechniqueHow a block is placedSearch cost
Direct mappingBlock → exactly one line, via (block address) mod (number of lines)Cheapest — one comparison
Fully associativeBlock → any line at allMost expensive — search all tags in parallel
Set-associative (k-way)Block → any of k lines within one specific setCompromise — search only k tags

For direct mapping, an address splits into three fields: Tag | Line (index) | Word (offset). Line bits = log₂(number of lines); word bits = log₂(words per block); tag bits = remaining address bits.

Worked example — address breakdown

16-bit address space, cache has 128 lines, block size = 8 words.

Word/offset bits = log₂8 = 3  |  Line bits = log₂128 = 7  |  Tag bits = 16 − 7 − 3 = 6

Average Memory Access Time (AMAT)

Key Formula
T_avg = h × T_cache + (1 − h) × T_main

where h = cache hit ratio. Worked example: T_cache = 10ns, T_main = 100ns, h = 0.9 → T_avg = 0.9×10 + 0.1×100 = 9 + 10 = 19ns.

Some questions instead use T_avg = T_cache + (1−h)×T_main (cache is always checked first, memory only on a miss) — read carefully to see which model the question assumes, since both appear in past papers.

Replacement & Write Policies

Replacement (needed when an associative/set-associative set is full): LRU (Least Recently Used), FIFO, LFU (Least Frequently Used), Optimal (theoretical best, used as a benchmark), and Random.

Write policies: write-through updates cache and main memory simultaneously (simple, always consistent, slower); write-back updates only the cache and marks the line "dirty," writing to main memory only when that line is evicted (faster, needs a dirty bit).

CA · 12 / 16Computer Architecture

Virtual Memory

Frequency

How a program addresses more memory than physically exists, and what happens when the part it needs isn't resident.

Paging

Logical address space is divided into fixed-size pages; physical memory into same-size frames. A page table maps page numbers to frame numbers. A logical address splits into page number | offset; the offset is copied unchanged into the physical address, while the page number is translated via the page table.

A TLB (Translation Lookaside Buffer) caches recent page-table entries so most translations skip the page-table lookup entirely.

Page Replacement Algorithms

FIFO (evict the oldest-loaded page), LRU (evict the least-recently-used page), and Optimal/MIN (evict the page that won't be needed for the longest time — theoretical, used only for comparison).

Common Trap — Belady's Anomaly

For FIFO specifically, increasing the number of available frames can sometimes increase the number of page faults — the opposite of what intuition suggests. This counterintuitive result, demonstrable on a specific reference string, is one of the most-repeated questions in this area. LRU and Optimal do not suffer from this anomaly.

Thrashing & Segmentation

Thrashing occurs when a process doesn't have enough frames to hold its working set — it spends more time paging data in and out than executing, and CPU utilisation collapses even though the CPU looks "busy."

Segmentation divides a program into logical, variable-sized units (code, data, stack), each described by a segment table entry giving a base address and a length/limit — unlike paging's fixed-size, purely physical division.

CA · 13 / 16Computer Architecture

I/O Organization

Frequency

Three escalating strategies for moving data between the CPU and peripheral devices.

Programmed I/O vs Interrupt-Driven vs DMA

MethodHow it worksCPU involvement
Programmed I/OCPU repeatedly polls the device status register (busy-waiting) until readyHighest — CPU does nothing else while waiting
Interrupt-driven I/OCPU continues other work; device raises an interrupt when readyModerate — CPU interrupted only when data is actually ready
DMAA dedicated DMA controller transfers entire blocks directly between device and memoryLowest — CPU involved only to initiate and at completion
Exam Focus

DMA and the CPU both need the system bus, so the DMA controller requests it and the CPU grants it (cycle stealing) — the CPU is briefly suspended for individual bus cycles, not for the entire transfer.

Interrupts

Maskable interrupts can be disabled by software; non-maskable cannot (reserved for critical events). Vectored interrupts let the device supply the address of its own service routine directly; with non-vectored interrupts the CPU jumps to a fixed address and must poll to find which device interrupted. With multiple devices, daisy-chaining or a priority encoder establishes interrupt priority.

CA · 14 / 16Computer Architecture

Pipelining & Hazards

Frequency

Highest-yield numerical topic in all of Computer Architecture — speedup, throughput, efficiency and hazard types appear in almost every NET paper.

What Pipelining Does

Pipelining overlaps the execution of multiple instructions by dividing execution into stages (e.g. Fetch, Decode, Execute, Write-back). While instruction N is in stage k, instruction N+1 is in stage k−1, N+2 in k−2, and so on — like an assembly line. The clock cycle = the longest single stage's latency (the bottleneck stage), not the sum of all stages.

Key Formulas

Formula 1 — Time with pipeline (k stages, n instructions, t = stage time)
T_pipeline = (k + n − 1) × t

Explanation: the first instruction takes k stages (k×t) to complete, then each subsequent instruction completes every t — so the remaining n−1 instructions add (n−1)×t.

Formula 2 — Speedup over sequential execution
Speedup = T_sequential / T_pipeline = (n × k × t) / ((k + n − 1) × t)

As n → ∞, speedup → k (the number of stages). In practice, hazards and stalls reduce this to well below k.

Formula 3 — Throughput & Efficiency
Throughput = n / T_pipeline      Efficiency = Throughput / (k × f) = Speedup / k

Efficiency ranges from 0 to 1 (or 0%–100%). When n is very large, efficiency approaches 100%.

Worked example

5-stage pipeline, 100 instructions, each stage 10 ns.

T_pipeline = (5 + 100 − 1) × 10 = 104 × 10 = 1040 ns

T_sequential = 100 × 5 × 10 = 5000 ns

Speedup = 5000 / 1040 ≈ 4.81  |  Efficiency = 4.81 / 5 ≈ 96.2%

Pipeline Hazards

Hazard typeCauseSolutions
StructuralTwo instructions need the same hardware resource at the same time (e.g. single memory for fetch + data access)Duplicate hardware (split I-cache and D-cache); stall one instruction
Data (RAW / WAR / WAW)An instruction needs data not yet produced by an earlier instruction in the pipelineForwarding/bypassing (RAW); stalls (bubbles); out-of-order execution; register renaming (WAR/WAW)
Control / BranchA branch changes the PC before the pipeline knows what to fetch nextBranch prediction; delayed branching; branch target buffer; flush on misprediction
Common Trap — RAW vs WAR vs WAW

RAW (Read After Write) — true data dependency, the most common, cannot be fully eliminated without forwarding or stalls. WAR (Write After Read) — anti-dependency, instruction writes before a preceding instruction reads; solved by register renaming. WAW (Write After Write) — output dependency; also solved by register renaming. RAW is the one that causes actual stalls in in-order pipelines.

Non-uniform Stage Latencies

When stages are unequal, the pipeline clock = the slowest stage. If the EX stage takes 30 ns while the others take 10 ns, the clock is 30 ns and the shorter stages sit idle during their remaining 20 ns. Normalising to the slow stage (or adding buffers to split it) is the design fix — and the basis of "what is the maximum pipeline throughput" questions that give unequal stage times.

Exam Focus

"Which property belongs to RISC?" is one of the most formulaic NET question types. The three RISC anchors that slip people up: fixed-length instructions (not shorter — CISC can also have short instructions, just not uniform ones), hardwired control unit, and load-store architecture (the only instructions that touch memory are LOAD and STORE; everything else is register-to-register).

The RISC vs CISC Debate in Practice

Modern x86 processors (CISC ISA) internally decode complex instructions into micro-operations and execute them on a RISC-like core — so the ISA boundary has become less meaningful at the micro-architecture level. However, the conceptual characteristics of each category are still clean and testable.

Common Trap

RISC does not mean "fewer transistors" or "simpler chip." The large register file and hardwired control logic of RISC chips can use more transistors than CISC chips. What's simpler is the instruction decode logic, not the chip overall.

Exam Focus

MISD is the one candidate that trips people up — it is not a common practical architecture. The standard exam answer is "fault-tolerant systems (same data processed by different instruction streams for cross-checking)" or "systolic arrays" depending on the text. Know it is mostly theoretical.

Amdahl's Law

Key Formula — Maximum Speedup
Speedup = 1 / ( (1 − P) + P/N )

where P = fraction of the program that can be parallelised, N = number of processors.

As N → ∞, speedup → 1/(1−P). So even with infinite processors, a program that is only 90% parallel (P=0.9) can achieve at most 10× speedup. The serial bottleneck (1−P) is the absolute ceiling. This is a favourite numerical question.

Worked example

P = 0.8, N = 4 → Speedup = 1 / (0.2 + 0.8/4) = 1 / (0.2 + 0.2) = 1/0.4 = 2.5×

Theoretical max (N→∞): 1/0.2 = 5×

Pipeline Formulas (k stages, n instructions, t = stage time)
T_pipeline = (k + n − 1) × t
